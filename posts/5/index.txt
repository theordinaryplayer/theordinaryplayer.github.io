10:T4227,<p class="markdown-p my-4 leading-relaxed">Open-Source Intelligence (OSINT) is the practice of collecting and analyzing publicly available information to gain insights into a target — be it an organization, infrastructure, or digital footprint. In cybersecurity, OSINT is critical for <strong class="markdown-strong font-bold">attack surface mapping, threat intelligence, vulnerability research</strong>, and <strong class="markdown-strong font-bold">incident response</strong>.</p>
<p class="markdown-p my-4 leading-relaxed"><img src="https://www.defenceonline.co.uk/wp-content/uploads/2024/11/osint-scaled.jpg" alt="1" class=""></p>
<p class="markdown-p my-4 leading-relaxed">This guide outlines how to conduct lawful and efficient OSINT for penetration testing, red teaming, or bug bounty reconnaissance, complete with safe tools and command-line workflows.</p>
<blockquote class="markdown-blockquote my-4 border-l-4 border-primary pl-4 italic text-muted-foreground">
<p class="markdown-p my-4 leading-relaxed"><strong class="markdown-strong font-bold">Note:</strong> All examples and commands are for <strong class="markdown-strong font-bold">authorized testing and research</strong> only. Never use these techniques against systems or individuals without explicit permission.</p>
</blockquote>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">1. Understanding OSINT</h2>
<p class="markdown-p my-4 leading-relaxed">OSINT isn’t just Googling a company’s name — it’s a structured intelligence process:</p>
<ol class="markdown-ol my-4 ml-6 list-decimal">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Define objective:</strong> What’s the mission? (e.g., identify subdomains or exposed assets)</li>
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Collect data:</strong> Use passive sources like archives, search engines, and public databases.</li>
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Process and enrich:</strong> Correlate data from multiple sources.</li>
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Analyze:</strong> Identify patterns, risks, and relationships.</li>
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Report:</strong> Present findings clearly, with actionable insights.</li>
</ol>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">2. Legal &amp; Ethical Framework</h2>
<p class="markdown-p my-4 leading-relaxed">OSINT operates within the boundaries of law and ethics. Key principles include:</p>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2">Collect only <strong class="markdown-strong font-bold">publicly available</strong> information.</li>
<li class="markdown-li my-2">Respect <strong class="markdown-strong font-bold">Terms of Service</strong> for all platforms.</li>
<li class="markdown-li my-2">Avoid scanning or brute-forcing without authorization.</li>
<li class="markdown-li my-2">Protect personal data — don’t disclose private information unless there’s a clear, lawful reason.</li>
<li class="markdown-li my-2">Always maintain an <strong class="markdown-strong font-bold">audit trail</strong> of your actions and evidence.</li>
</ul>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">3. OSINT Toolkit (with Real Commands)</h2>
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">Web &amp; Archive Discovery</h3>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">gau</strong> – Retrieve archived URLs from sources like Wayback Machine:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">gau example.com &gt; urls.txt
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">waybackurls</strong> – Get historical snapshots:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">waybackurls example.com &gt; wayback.txt
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">curl</strong> – Download and inspect a page:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">curl -sL <span class="hljs-string">'https://example.com/page'</span> -o page.html
</code></pre>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">Subdomain &amp; DNS Enumeration</h3>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">amass</strong> – Comprehensive subdomain discovery:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">amass enum -d example.com -o amass.txt
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">subfinder</strong> – Fast passive subdomain finder:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">subfinder -d example.com -o subs.txt
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">dig</strong> – Check DNS records:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">dig +short example.com ANY
</code></pre>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">Passive Service Discovery</h3>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Shodan</strong> – Search internet-exposed devices:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">shodan search --fields ip_str,port,org <span class="hljs-string">"hostname:example.com"</span>
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Censys</strong> – Identify hosts via certificate and banner data.</li>
</ul>
<blockquote class="markdown-blockquote my-4 border-l-4 border-primary pl-4 italic text-muted-foreground">
<p class="markdown-p my-4 leading-relaxed">Passive reconnaissance avoids triggering alarms or violating access policies.</p>
</blockquote>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">Email, Identity &amp; Leak Search</h3>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">theHarvester</strong> – Gather emails, hosts, and subdomains:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">theHarvester -d example.com -b all -l 500
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">HaveIBeenPwned API</strong> – Check if emails appear in breaches.</li>
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Hunter.io</strong> – Find corporate email patterns and contacts.</li>
</ul>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">URL Filtering and Validation</h3>
<p class="markdown-p my-4 leading-relaxed">Use <code class="markdown-code font-mono text-sm bg-muted px-1.5 py-0.5 rounded">gf</code> and <code class="markdown-code font-mono text-sm bg-muted px-1.5 py-0.5 rounded">httpx</code> to detect live endpoints with interesting parameters:</p>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm"><span class="hljs-built_in">cat</span> urls.txt | gf lfi | httpx -threads 50 -o live_lfi.txt
</code></pre>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">Social Media &amp; People Search</h3>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">LinkedIn, GitHub, Twitter, Facebook</strong> – Manual and API-based searches.</li>
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">Maltego</strong> – Visual link analysis between identities, domains, and organizations.</li>
<li class="markdown-li my-2"><strong class="markdown-strong font-bold">SpiderFoot</strong> – Automated OSINT platform for aggregation and enrichment.</li>
</ul>
<blockquote class="markdown-blockquote my-4 border-l-4 border-primary pl-4 italic text-muted-foreground">
<p class="markdown-p my-4 leading-relaxed">Always use dummy or research accounts. Never engage with targets directly.</p>
</blockquote>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">4. Google Dorking for Discovery</h2>
<p class="markdown-p my-4 leading-relaxed">Example Google queries (dorks) for reconnaissance:</p>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2">Find admin panels:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-vbnet markdown-code font-mono text-sm"><span class="hljs-symbol">site:</span>example.com intitle:<span class="hljs-string">"login"</span>
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2">Find configuration files:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">site:example.com ext:conf OR ext:<span class="hljs-built_in">env</span> OR ext:ini
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2">Find exposed documents:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-makefile markdown-code font-mono text-sm"><span class="hljs-section">site:example.com filetype:pdf OR filetype:xlsx</span>
</code></pre>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2">GitHub code exposure:</li>
</ul>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-vbnet markdown-code font-mono text-sm"><span class="hljs-symbol">org:</span>example <span class="hljs-string">"AWS_SECRET_ACCESS_KEY"</span>
</code></pre>
<p class="markdown-p my-4 leading-relaxed"><strong class="markdown-strong font-bold">Use responsibly.</strong> Avoid accessing or downloading sensitive files from external sources.</p>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">5. Certificate Transparency &amp; Domain Correlation</h2>
<p class="markdown-p my-4 leading-relaxed">Use <strong class="markdown-strong font-bold">crt.sh</strong> to find hidden subdomains:</p>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">curl -s <span class="hljs-string">"https://crt.sh/?q=%25.example.com&amp;output=json"</span> | jq -r <span class="hljs-string">'.[].name_value'</span>
</code></pre>
<p class="markdown-p my-4 leading-relaxed">Use the output to feed enumeration tools like <code class="markdown-code font-mono text-sm bg-muted px-1.5 py-0.5 rounded">amass</code> or <code class="markdown-code font-mono text-sm bg-muted px-1.5 py-0.5 rounded">subfinder</code>.</p>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">6. Timeline Reconstruction</h2>
<p class="markdown-p my-4 leading-relaxed">When conducting threat or leak analysis:</p>
<ol class="markdown-ol my-4 ml-6 list-decimal">
<li class="markdown-li my-2">Record timestamps (archive date, WHOIS updates, commit history).</li>
<li class="markdown-li my-2">Normalize to UTC (ISO 8601 format).</li>
<li class="markdown-li my-2">Correlate key events — e.g., domain registration → GitHub repo creation → credential leak.</li>
<li class="markdown-li my-2">Store immutable evidence (hash files, archive URLs).</li>
</ol>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">7. OPSEC for OSINT Investigators</h2>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2">Use <strong class="markdown-strong font-bold">VPNs or dedicated research environments</strong> (VMs, containers).</li>
<li class="markdown-li my-2">Isolate browser sessions per target.</li>
<li class="markdown-li my-2">Avoid personal accounts or cookies.</li>
<li class="markdown-li my-2">Maintain anonymous accounts for testing (within legal limits).</li>
<li class="markdown-li my-2">Never “like,” “follow,” or interact with targets under observation.</li>
</ul>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">8. Common OSINT Workflows</h2>
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">A. Find Public S3 Buckets</h3>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">grep -Eo <span class="hljs-string">"s3\.amazonaws\.com/[^\"' ]+"</span> urls.txt | <span class="hljs-built_in">sort</span> -u
</code></pre>
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">B. Enumerate Subdomains via Certificates</h3>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">curl -s <span class="hljs-string">"https://crt.sh/?q=%25.example.com&amp;output=json"</span> | jq -r <span class="hljs-string">'.[].name_value'</span>
</code></pre>
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">C. Shodan for Open Ports</h3>
<pre class="markdown-pre rounded-md bg-muted p-4 overflow-x-auto"><code class="hljs language-bash markdown-code font-mono text-sm">shodan search --fields ip,port,org <span class="hljs-string">"ssl.cert.subject.CN:example.com"</span>
</code></pre>
<h3 class="markdown-h3 text-xl font-bold mt-6 mb-4">D. Collect Paste Leaks (manual or via APIs)</h3>
<p class="markdown-p my-4 leading-relaxed">Use paste search engines or breach monitoring services — not public dumps.</p>
<hr class="markdown-hr my-6 h-px border-0 bg-border">
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">9. OSINT Reporting Template</h2>
<p class="markdown-p my-4 leading-relaxed"><strong class="markdown-strong font-bold">Title:</strong> Exposed Backup File on Public Server
<strong class="markdown-strong font-bold">Scope:</strong> example.com (and related subdomains)
<strong class="markdown-strong font-bold">Impact:</strong> Publicly accessible SQL backup containing user data.
<strong class="markdown-strong font-bold">Evidence:</strong> URL, screenshots, timestamps, SHA256 hash of downloaded file.
<strong class="markdown-strong font-bold">Recommendation:</strong> Restrict directory listing, remove backup, rotate database credentials.</p>
<h2 class="markdown-h2 text-2xl font-bold mt-6 mb-4">10. Key Red Flags</h2>
<ul class="markdown-ul my-4 ml-6 list-disc">
<li class="markdown-li my-2">Publicly accessible credentials (<code class="markdown-code font-mono text-sm bg-muted px-1.5 py-0.5 rounded">.env</code>, <code class="markdown-code font-mono text-sm bg-muted px-1.5 py-0.5 rounded">.git</code>, or config files).</li>
<li class="markdown-li my-2">Leaked databases or S3 buckets.</li>
<li class="markdown-li my-2">Exposed admin panels or dev subdomains.</li>
<li class="markdown-li my-2">Sensitive information on GitHub (tokens, private repos).</li>
</ul>
<p class="markdown-p my-4 leading-relaxed">Treat these as <strong class="markdown-strong font-bold">high-priority disclosures</strong> — report via official bug bounty or responsible disclosure channels.</p>
